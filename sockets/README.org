** Exercise 16.6
:PROPERTIES:
:CUSTOM_ID: exercise-166
:END:

Write two library routines: one to enable asynchronous (signal-based) I/O on a socket and
one to disable asynchronous I/O on a socket. Use Figure 16.23 to make sure that the
functions work on all platforms with as many socket types as possible.

*** Solution
:PROPERTIES:
:CUSTOM_ID: solution-16-6
:END:

To enable asynchronous socket I/O, we need to establish socket
ownership using the F_SETOWN fcntl command, and then enable
asynchronous signaling using the FIOASYNC ioctl command. To disable
asynchronous socket I/O, we simply need to disable asynchronous
signaling. The reason we mix fcntl and ioctl commands is to find the
methods that are most portable. The code is shown in [[file:../exercises/asyncsocket.c][Figure C.23]].
