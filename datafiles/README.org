** Exercise 6.1
:PROPERTIES:
:CUSTOM_ID: exercise-61
:END:

If the system uses a shadow file and we need to obtain the encrypted
password, how do we do so?

** Exercise 6.2
:PROPERTIES:
:CUSTOM_ID: exercise-62
:END:

If you have superuser access and your system uses shadow passwords,
implement the previous exercise.

*** Solution
:PROPERTIES:
:VISIBILITY: folded
:END:

The program in [[file:../exercises/getpwsvr4.c][Figure C.5]] prints the encrypted password on *Linux
3.2.0* and *Solaris 10*. Unless this program is run with superuser
permissions, the call to =getspnam= fails with an error of =EACCES=.

Under *FreeBSD 8.0*, the program in [[file:../exercises/getpw44bsd.c][Figure C.6]] prints the encrypted
password if the program is run with superuser permissions. Otherwise,
the value returned in =pw_passwd= is an asterisk. On *Mac OS X
10.6.8*, asterisks are printed regardless of the permissions with
which it is run.

** Exercise 6.5
:PROPERTIES:
:CUSTOM_ID: exercise-65
:END:

Write a program to obtain the current time and print it using
strftime, so that it looks like the default output from date(1). Set
the TZ environment variable to different values and see what happens.

*** Solution

The program shown in [[file:../exercises/prtime.c][Figure C.7]] prints the date in a format similar to
the date command.

Running this program gives us

#+begin_src sh
$ ./a.out                       # author ’s default is US/Eastern
Wed Jul 25 22:58:32 EDT 2012

$ TZ=US/Mountain ./a.out        # U.S. Mountain time zone
Wed Jul 25 20:58:32 MDT 2012

$ TZ=Japan ./a.out              # Japan
Thu Jul 26 11:58:32 JST 2012
#+end_src
