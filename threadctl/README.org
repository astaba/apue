** Exercise 12.6
:PROPERTIES:
:CUSTOM_ID: exercise-126
:END:

Reimplement the program in Figure 10.29 to make it thread-safe without using
nanosleep or clock_nanosleep.

*** Solution
:PROPERTIES:
:CUSTOM_ID: solution-12-6
:END:

[[file:../exercises/sleep.c][Figure C.13]] shows a thread-safe sleep implementation that uses select
to delay for the specified amount of time. It is thread-safe because
it doesn’t use any unprotected global or static data and calls only
other thread-safe functions.
